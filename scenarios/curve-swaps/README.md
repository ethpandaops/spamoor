# Curve Swaps

Execute Curve StableSwap exchanges against self-deployed 3-coin stable pools. This scenario stress-tests the compute-heavy StableSwap invariant math (the Newton's-method `get_D` / `get_y` iterations) that makes Curve one of Ethereum's heaviest gas consumers.

The scenario deploys the **canonical Curve `StableSwap` 3pool and `CurveTokenV3` LP token (real Vyper bytecode)**, so the on-chain behavior and gas profile match mainnet Curve rather than a reimplementation. On a fresh network it self-deploys everything it needs: one set of three 18-decimal mock stablecoins per pool, the LP token, the StableSwap pool, and a liquidity provider helper, then seeds balanced liquidity. Two minimal edits are applied to the vendored Curve sources (documented in their file headers): the coin rate multipliers are set for 18-decimal mock coins, and the LP token takes its minter as a constructor parameter (because contracts are deployed via a CREATE2 factory).

Each child wallet is funded with the first coin of every pool and approves the pools. To keep both wallets and pools balanced, every swap sells the coin the wallet holds most of into a random other coin.

## Usage

```bash
spamoor curve-swaps [flags]
```

## Configuration

### Base Settings (required)
- `--privkey` - Private key of the sending wallet
- `--rpchost` - RPC endpoint(s) to send transactions to

### Volume Control (either -c or -t required)
- `-c, --count` - Total number of transactions to send
- `-t, --throughput` - Transactions to send per slot
- `--max-pending` - Maximum number of pending transactions

### Transaction Settings
- `--basefee` - Max fee per gas in gwei (default: 20)
- `--tipfee` - Max tip per gas in gwei (default: 2)
- `--basefee-wei` / `--tipfee-wei` - Fee/tip per gas in wei (overrides the gwei flags for L2 sub-gwei fees)
- `--rebroadcast` - Enable the reliable rebroadcast system (default: 1)

### Pool Settings
- `--pool-count` - Number of StableSwap pools to deploy (default: 1)
- `--amplification` - StableSwap amplification coefficient A (default: 200)
- `--fee` - Swap fee, denominated in 1e10 (default: 4000000 = 0.04%)
- `--seed-amount` - Liquidity seeded per coin into each pool, in wei (default: 1000000000000000000000000 = 1,000,000 tokens)

### Swap Settings
- `--wallet-funding` - Initial coin balance minted to each wallet, in wei (default: 10000000000000000000000 = 10,000 tokens)
- `--min-swap` - Minimum swap amount in wei (default: 1000000000000000000 = 1 token)
- `--max-swap` - Maximum swap amount in wei (default: 1000000000000000000000 = 1,000 tokens)
- `--slippage` - Slippage tolerance in basis points (default: 100)

### Wallet Management
- `--max-wallets` - Maximum number of child wallets to use
- `--refill-amount` - ETH amount to fund each child wallet (default: 5)
- `--refill-balance` - Minimum ETH balance before refilling (default: 2)
- `--refill-interval` - Seconds between balance checks (default: 300)

### Client Settings
- `--client-group` - Client group to use for sending transactions
- `--deploy-client-group` - Client group to use for deployment transactions (same as --client-group if empty)

### Debug Options
- `-v, --verbose` - Enable verbose output
- `--log-txs` - Log all submitted transactions
- `--trace` - Enable tracing output

## Examples

Send 100 swaps across a single pool:
```bash
spamoor curve-swaps -p "<PRIVKEY>" -h http://rpc-host:8545 -c 100
```

Send 5 swaps per slot across 3 pools:
```bash
spamoor curve-swaps -p "<PRIVKEY>" -h http://rpc-host:8545 -t 5 --pool-count 3
```

Use a higher amplification coefficient (flatter curve, closer to constant-sum):
```bash
spamoor curve-swaps -p "<PRIVKEY>" -h http://rpc-host:8545 -t 5 --amplification 2000
```

## Contracts

The compiled Go bindings under `contract/` are generated by `contract/compile.sh` (Docker-based solc + Vyper + abigen):

- `StableSwap.vy` / `CurveToken.vy` - canonical Curve 3pool and LP token (Vyper 0.2.x), with the two minimal spamoor edits noted in their headers.
- `MintableToken.sol` - minimal 18-decimal mock stablecoin with a permissionless `mint`.
- `CurveLiquidityProvider.sol` - seeds a pool with balanced liquidity in a single transaction.
